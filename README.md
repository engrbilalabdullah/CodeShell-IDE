# NexusShell IDE


A high-performance, single-file custom Integrated Development Environment (IDE) built from scratch using **C++ (SFML 3)** for the GUI dashboard and **Windows API** for low-level operating system operations. 

This project was developed independently to demonstrate advanced concepts of process creation, inter-process communication (IPC), and real-time GUI window rendering.

---

## 🖥️ Project Dashboard Layout

* **Left Split Panel:** Features a dynamic Virtual File Explorer Tree and a scrollable Hacker-Green Theme Terminal.
* **Right Split Panel:** A responsive, high-contrast Code Editor supporting fully flexible mouse-click cursor placement.
* **Adjustable UI System:** Both panels are separated by dynamic vertical and horizontal splitters that can be **dragged with the mouse** to resize the layout in real time.

---

## 🛠️ Core OS Concepts Implemented

This IDE goes beyond standard text editing by interacting directly with the Windows Kernel to simulate an operational OS shell environment:

1. **Process Management (`CreateProcessA`)**: Instead of using risky high-level commands like `system()`, the IDE directly requests the OS Kernel to launch low-level child processes (`g++.exe` for C++ compilation and `python.exe` for scripts) running silently in the background.
2. **Inter-Process Communication (IPC via Anonymous Pipes)**: Implemented `CreatePipe` to setup unidirectional communication channels. The output (`stdout` and `stderr`) generated by the background compiler or runtime engine is captured through these pipes and redirected straight into the IDE's terminal buffer.
3. **System Clipboard Integration**: Integrated the OS Clipboard Manager to seamlessly support multi-line `Ctrl + V` and **Mouse Right-Click Paste** mechanics without locking or crashing the main render thread.
4. **Viewport Clipping & Rendering**: Utilized standard `sf::View` slicing models to clip overflowing code texts inside strict vector layouts, preventing graphic overlaps.

## Compilation & Build Setup

To build this engine on your local machine, ensure you have **SFML 3** configured inside your compiler search directories.

### Compiler Flag Configuration (Crucial)
Since the project relies heavily on the clean modern C++ standard library mechanics (like `std::clamp` and `<optional>`), you **MUST** enable **C++17** or higher inside your compiler flags.

```bash
# Terminal compilation command string via MinGW g++
g++ main.cpp -o CodeShell_IDE.exe -lsfml-graphics -lsfml-window -lsfml-system -std=c++17
